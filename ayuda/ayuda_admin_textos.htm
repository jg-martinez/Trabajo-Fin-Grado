Ayuda de Calíope

Administración de Textos

última actualización:
21 de septiembre de 2010

 

1. Menú de administración de textos

El menú de administración de textos presenta por pantalla un buscador con los textos existentes en la base de datos. De cada texto se listan los siguientes datos:

  • título: es el título del texto; va precedido de un número entre corchetes [X], que es un identificador único para cada texto
  • número de palabras
  • idioma
  • tipo del texto
  • campo del texto

Cuando alguno de estos datos se desconozca, la casilla correspondiente de la lista permanecerá en blanco. 

Las operaciones de administración disponibles son las siguientes:

  • altade un texto nuevo,
  • eliminación de un texto,
  • modificación de un texto.

 

2. Operaciones

En la parte izquierda de la pantalla aparece una columna con algunas de las operaciones disponibles para la administración de los textos y otras operaciones:

® Creación de un texto nuevo
® Vuelve al menú principal de administración de textos

 La administración de fuentes de texto se completa con los dos iconos que aparecen en la lista de fuentes de texto, al final de cada línea :

 ® Modificación de losdatos de un texto

® Eliminación de un texto

 

2.1. Alta de un texto nuevo

Para dar de alta un nuevo texto hay que pulsar el botón del menú de administración de textos que se indica en el apartado 2. Operaciones .

Es necesario rellenar el formulario con los datos del texto. Hay dos campos que hay rellenar de forma obligatoria y sin los cuales no se puede dar de alta un texto nuevo: el título del texto y el fichero en el que está almacenado . Si estos datos no se introducen, el sistema muestra un mensaje de aviso por pantalla.

El título del texto no tiene que ser único, puede haber varios textos con el mismo título. Por eso, para identificar cada texto de forma unívoca, el sistema asigna de forma automática un identificador único para cada texto cuando es dado de alta (es el identificador [X] que aparece en el listado de textos del menú de administración de textos).

También se puede relacionar con otro texto existente mediante el campo correspondiente.

El sistema cuenta el número de palabras del texto y el número de bytes que ocupa el fichero, y guarda esta información automáticamente en la base de datos.

 

2.2. Eliminación de un texto

Tras pulsar el botón , se muestra en pantalla un mensaje de confirmación de la eliminación junto con los datos del texto que se va a eliminar. Esta confirmación es necesaria para asegurarnos que se elimina el texto deseado y no otro cualquiera que se podría haber seleccionado por error.

En caso de que exista en el glosario algún término con contextos pertenecientes al texto que se quiere eliminar, el sistema mostrará por pantalla un mensaje de aviso informando al usuario de este hecho . La eliminación en este caso sí continuará, eliminándose del glosario dichos contextos (no se eliminará el término al que pertenece el contexto, que seguirá almacenado en la base de datos).

 

2.3. Modificación de un texto

Tras pulsar el botón , se muestra en pantalla un formulario con todos los datos del texto seleccionado. Todos los campos son modificables, excepto el formato del fichero en el que está alojado el texto, el número de bytes que ocupa y el número de palabras del texto (estos datos no cambian nunca una vez se ha introducido el fichero en la base de datos).

Después de haber modificado los datos, se confirman pulsando el botón [ Aceptar ].
