Administración de Usuarios

1. Menú de administración de usuarios

El menú de administración de usuarios presenta por pantalla una lista con los datos de todos los usuarios que pueden acceder al sistema. La lista está dividida en dos partes: los usuarios con privilegios de administrador, y los usuarios con privilegios de usuario.

De cada usuario se listan los siguientes datos:

  • login (nombre de usuario, que es único para cada usuario del sistema)
  • DNI / número de matrícula (para alumnos de la Facultad de Informática)
  • nombre y apellidos
  • email

Cuando alguno de estos datos se desconozca, la casilla correspondiente de la lista permanecerá en blanco. Por razones de seguridad, la contraseña de los usuarios no se muestra por pantalla.

Las operaciones de administración disponibles son las siguientes:

  • alta de nuevo usuario,
  • alta de nuevos usuarios (a partir de un fichero), 
  • cambio de contraseña
  • eliminación de un usuario,
  • modificación de un usuario.

2. Operaciones

En la parte izquierda de la pantalla aparece una columna con algunas de las operaciones disponibles para la administración de los usuarios:

 ® Creación de un usuario nuevo

® Creación de un conjunto de usuarios nuevos a partir de los datos alojados en un fichero

 La administración de usuarios se completa con los dos iconos que aparecen en la lista de usuarios, al final de cada línea :

 ® Modificación de losdatos de un usuario

® Modificación de contraseña

® Eliminación de un usuario

2.1. Alta de un usuario nuevo

Para dar de alta un nuevo usuario hay que pulsar el botón del menú de administración de usuarios que se indica en el apartado 2. Operaciones .

Es necesario rellenar el formulario con los datos del usuario. Hay varios campos que hay rellenar de forma obligatoria y sin los cuales no se puede dar de alta un nuevo usuario: login y contraseña (password). Este último dato hay que introducirlo de forma repetida para evitar que el usuario pueda equivocarse a la hora de escribirlo. Si alguno de estos datos no se introducen, el sistema muestra un mensaje de aviso por pantalla.

Como es evidente, el login de cada usuario es único. Por tanto si se intenta introducir un usuario nuevo cuyo login ya es propiedad de un usuario existente, el sistema avisará del error y no se procederá con el alta de usuario.

2.2. Alta de usuarios nuevos (a partir de un fichero)

Mediante esta opción, se pueden dar de alta varios usuarios a la vez. Los datos de estos nuevos usuarios deben estar contenidos en un fichero de texto que el usuario debe proporcionar al sistema pulsando el correspondiente botón [Examinar...] que aparece en la ventana.

El fichero debe contener el login de los usuarios que se desean dar de alta, uno por cada línea del fichero. Un ejemplo de fichero válido sería el siguiente:

raul_barahona
pedro_lopez
d960252
beatriz

El único dato proporcionado para cada usuario nuevo es el login, por lo que la contraseña de cada uno de ellos coincidirá con su contraseña. Esta opción se ha implementado de esta manera debido a que, en principio, la aplicación será usada casi totalmente por alumnos de la Facultad de Informática (UPM), y ésta es una manera rápida y sencilla para dar de alta de una sola vez a un conjunto de alumnos.

El sistema verificará que los login incluídos en el fichero no pertenecen a ningún usuario ya existente. Se mostrará por pantalla la confirmación de la validez de los datos del fichero:

login_del_usuario

 ® El usuario se ha dado de alta correctamente

login_del_usuario

 ® El usuario no se ha dado de alta, el login es propiedad de un usuario ya existente

2.3. Eliminación de un usuario

Tras pulsar el botón , se muestra en pantalla un mensaje de confirmación de la eliminación junto con los datos del usuario que se va a eliminar. Esta confirmación es necesaria para asegurar que se elimina el usuario deseado y no otro cualquiera que se podría haber seleccionado por error.

2.4. Modificación de contraseña

Tras pulsar el botón , se muestra una pantalla con dos campos de texto para introducir la nueva contraseña y su repetición. Esto permite que una contraseña no se introduzca por error.

2.5. Modificación de un usuario

Tras pulsar el botón , se muestra en pantalla un formulario con todos los datos del usuario seleccionado. Todos los campos son modificables excepto el login y la contraseña, para evitar errores de coherencia.

Después de haber modificado los datos, se confirman pulsando el botón [ Aceptar ].
